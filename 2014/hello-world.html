初识 Git

早就听说过Git的威名了，真正开始陆陆续续接触也就是这两个月的事情。

写Paper的时候发现，做一点修改就要备份一次，相当的蹉跎和不方便。接触到Git发现，这玩意儿就是为版本控制而生的，而且远比我们想象中的强大。比如，这个博客，也是建立在Github上面的。

顺便推荐一个好玩的Git教程。

再说 Hexo

一开始搭建Github Pages, 使用的Jekyll, 够折腾的，部署网站非常麻烦。而且对官网的文档介绍，觉得不满意，写得很生涩。

技术文档简单介绍理论知识能举例子说明是最好的。这方面的典范当推Mathematica的说明文档了，后续会在这个博客上跟大家分享Mathematica的妖娆美丽。

之后呢，又试着用Simple搭建了一个博客，易用性很好，但默认主题不是很喜欢，也难得再去折腾CSS了。

于是投靠了Hexo, 一位台湾同学基于Node.js搭建的，默认主题大气，其余可供选择的漂亮主题模板也多，比如Light, Pacman等。Hexo说明文档清晰易懂。

最后谈谈 MarkDown

这门轻量级标记语言非常给力啊，简单易懂，基本你看完上面的维基百科就知道怎么用了。

现在一般用它来写我的实验记录，因为小巧，可以在移动端方便的查看，随时翻阅。大型分析报告一般用Mathematica写，写Paper自然是用LaTeX。

基于Markdown比较又代表的工具有：简书和为知笔记。
